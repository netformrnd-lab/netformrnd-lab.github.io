# 모여라딜 어드민 시스템 - 종합 UX 리뷰 보고서

## 1. 전체 메뉴 구조 분석

### 1.1 현재 메뉴 구조

**운영센터 (admin/)**
```
📊 영업현황 ← 핵심 대시보드
공구센터
  ├─ 📅 캘린더
  ├─ ✅ 공구 현황/체크리스트
  └─ 💡 공구 제안 관리
파트너 관리
  ├─ 공급사 관리 (공급사 DB / 상품 DB / 상품 리스트)
  ├─ 셀러 관리 (셀러 DB / 셀러 리스트)
  ├─ 파트너 계정관리
  └─ 파트너 공지
🛒 카페24 주문 공유 [파트너용]
💰 카페24 정산 [정산서용]
  ├─ 1️⃣ 엑셀 업로드 & 매핑
  ├─ 2️⃣ 주문 관리
  └─ 3️⃣ 정산서 생성 & 관리
📚 업무 가이드
관리자 🔐
```

**전략센터 (admin2/)**
```
🎯 그로스보드
  ├─ 📖 사용 가이드
  ├─ ① 전략기획
  ├─ ② KPI 관리
  ├─ ③ 업무일지
  ├─ ④ 현황 대시보드
  └─ ⑤ 분기 리뷰
📚 레퍼런스
관리자 🔐
```

## 2. 메뉴 구조 문제점 및 개선안

### 2.1 불필요하거나 겹치는 메뉴

| 문제 | 현재 상태 | 개선안 |
|------|----------|--------|
| **공급사 DB vs 상품 DB vs 상품 리스트** | 3개 메뉴가 유사한 기능 수행 | `공급사 관리` 하나로 통합, 탭으로 구분 |
| **셀러 DB vs 셀러 리스트** | 2개 메뉴가 중복 | `셀러 관리` 하나로 통합, 탭으로 구분 |
| **운영센터/전략센터 분리** | 운영 흐름이 끊김 | 통합 또는 퀵 네비게이션 강화 |
| **영업현황 (운영) vs 현황 대시보드 (전략)** | 유사 대시보드가 분리됨 | 통합 대시보드로 일원화 |

### 2.2 빠진 기능 (운영 흐름 기준)

| 운영 단계 | 빠진 기능 | 필요 이유 |
|----------|----------|----------|
| **공구 생성** | 빠른 공구 생성 버튼 | 캘린더에서 바로 공구 등록 필요 |
| **공구 진행** | 공구별 실시간 주문 현황 | 현재 카페24 주문과 공구가 분리됨 |
| **정산** | 정산 예정 알림 | D+7 넘어가면 자동 알림 필요 |
| **관리** | 전체 검색 기능 | 공급사/셀러/공구 통합 검색 없음 |

### 2.3 메뉴 명칭 개선안

| 현재 명칭 | 문제점 | 개선안 |
|----------|--------|--------|
| 공구 현황/체크리스트 | 이름이 길고 모호함 | `공구 관리` |
| 공구 제안 관리 | 제안의 주체가 불명확 | `신규 공구 제안` |
| 카페24 주문 공유 | 카페24가 내부 용어 | `주문 현황` |
| 카페24 정산 | 외부 시스템명 노출 | `정산 관리` |
| 파트너 계정관리 | 무엇을 관리하는지 불명확 | `파트너 로그인 관리` |

## 3. UX 관점 개선 사항 (구체적 제안)

### 3.1 클릭 동선이 불필요하게 긴 부분

#### 문제 1: 정산서 생성까지 5단계
**현재:** 정산 메뉴 → 엑셀 업로드 모달 → 업로드 → 매핑 → 정산서 생성 버튼 → 정산서 확인
**개선안:**
```
정산 메뉴 진입 시 자동으로 "미정산 공구" 목록 표시
→ 공구 선택 → 원클릭 정산서 생성
```

#### 문제 2: 공구 생성 시 공급사/셀러 선택
**현재:** 공구 등록 → 공급사 드롭다운 → (없으면) 공급사 DB 이동 → 등록 → 다시 공구 등록
**개선안:**
```
공구 등록 모달 내에서 공급사 검색 + 빠른 신규 등록 인라인 지원
```

#### 문제 3: 운영센터 ↔ 전략센터 이동
**현재:** 사이드바 상단 링크 클릭 → 새 페이지 로딩
**개선안:**
```
상단 헤더에 센터 전환 탭 배치
또는 FAB(플로팅 버튼)으로 빠른 이동
```

### 3.2 한 화면에서 함께 보여주면 좋은 정보

#### 제안 1: 공구 상세 화면 통합
현재 분리된 정보를 한 화면에:
```
┌─────────────────────────────────────────────────┐
│ 공구명: [ㅇㅇ 공구]                    D-Day: 3일차 │
├─────────────────────────────────────────────────┤
│ 📦 주문 현황              │ ✅ 체크리스트           │
│ ├─ 총 주문: 152건         │ ├─ [x] 샘플 전달       │
│ ├─ 총 매출: 2,340,000원   │ ├─ [x] 결제링크 전달   │
│ └─ 목표 달성률: 78%       │ ├─ [ ] 콘텐츠 전달     │
│                          │ └─ [ ] 광고 세팅       │
├─────────────────────────────────────────────────┤
│ 💰 정산 예상              │ 📞 파트너 정보         │
│ ├─ 셀러 지급: 1,600,000원 │ ├─ 공급사: ㅇㅇ푸드    │
│ ├─ 공급사 정산: 800,000원 │ ├─ 셀러: 홍길동 (인스타)│
│ └─ 순이익: 약 140,000원   │ └─ 담당: 김송희        │
└─────────────────────────────────────────────────┘
```

#### 제안 2: 영업현황 대시보드에 액션 버튼 추가
```
현재: 숫자만 표시
개선: 숫자 클릭 시 해당 상세 화면으로 이동 + 퀵 액션 버튼
예) "이번 주 공구 3건" 카드 → 클릭하면 해당 공구 목록 + "새 공구 등록" 버튼
```

### 3.3 자주 쓰는 기능 빠른 접근 구조

#### 현재 사용 빈도 높은 기능 (추정)
1. 공구 현황 체크리스트 확인/업데이트
2. 정산서 생성
3. 주문 현황 확인
4. 새 공구 등록

#### 개선안: 글로벌 퀵 액션 바
```
┌─────────────────────────────────────────────────────────────┐
│ [+ 새 공구] [📋 체크리스트] [💰 정산] [📊 현황] [🔍 검색]   │
└─────────────────────────────────────────────────────────────┘
```
- 모든 화면 상단에 고정
- 아이콘 + 텍스트로 직관성 확보
- 현재 화면의 컨텍스트 유지하면서 모달로 빠른 작업

## 4. UI/디자인 불일관성 분석

### 4.1 컬러 시스템 불일관

| 화면 | 현재 사용 컬러 | 문제점 |
|------|--------------|--------|
| 영업현황 | 파랑 그라디언트 (#1e40af → #3b82f6) | 주황 브랜드 컬러와 충돌 |
| 정산 관리 | 초록 그라디언트 (#059669 → #10b981) | 섹션마다 색상 다름 |
| 주문 공유 | 파랑 그라디언트 (#2563eb → #3b82f6) | 영업현황과 유사하지만 미묘하게 다름 |
| 전략기획 | 주황 (#fc9600), 노랑 (#FEF3C7) | 브랜드 컬러와 일치 ✓ |
| 그로스보드 | 초록 (#10b981 → #059669) | 전략센터 자체 컬러 |

### 4.2 버튼 스타일 불일관

**발견된 버튼 유형:**
1. `background: var(--gray-100)` - 회색 약한 버튼
2. `background: var(--gray-900)` - 검정 강한 버튼
3. `background: linear-gradient(...)` - 그라디언트 버튼 (색상 제각각)
4. `border: 2px solid ...` - 아웃라인 버튼

**문제:** 같은 중요도의 액션이 화면마다 다른 스타일

### 4.3 카드/섹션 스타일 불일관

| 스타일 | 사용 화면 |
|--------|----------|
| `border-radius: 16px` + `box-shadow` | 전략기획, 영업현황 |
| `border-radius: 12px` + `border: 1px solid` | 정산 관리 |
| `border-radius: 8px` | 일부 버튼, 입력 필드 |

## 5. 전략기획 메뉴 기준 디자인 가이드

### 5.1 레퍼런스 스타일 (전략기획 메뉴)

```css
/* 카드 기본 */
background: white;
border-radius: 16px;
padding: 24px;
box-shadow: 0 2px 8px rgba(0,0,0,0.06);

/* KPI 카드 */
background: linear-gradient(135deg, #fc9600, #e08600);
border-radius: 16px;
padding: 20px;
color: white;

/* 강조 섹션 (성공률 등) */
background: linear-gradient(135deg, #FEF3C7, #FDE68A);
border: 2px solid #F59E0B;
border-radius: 16px;

/* 프로그레스 바 */
background: var(--gray-200);
height: 5px;
border-radius: 3px;

/* 버튼 - 기본 */
background: var(--gray-100);
border: none;
padding: 10px 20px;
border-radius: 8px;
font-size: 13px;
font-weight: 600;

/* 버튼 - 강조 */
background: var(--primary);  /* #fc9600 */
color: white;
border: none;
padding: 10px 20px;
border-radius: 8px;
```

### 5.2 통일해야 할 스타일 규칙

| 요소 | 통일 규칙 |
|------|----------|
| **카드** | `border-radius: 16px`, `padding: 24px`, `box-shadow: 0 2px 8px rgba(0,0,0,0.06)` |
| **KPI 카드** | 주황 그라디언트 `linear-gradient(135deg, #fc9600, #e08600)` |
| **강조 배지** | 노랑 배경 `#FEF3C7`, 갈색 텍스트 `#92400E` |
| **버튼 기본** | `background: var(--gray-100)`, `border-radius: 8px` |
| **버튼 강조** | `background: var(--primary)`, `color: white` |
| **프로그레스** | 높이 5-6px, `border-radius: 3px` |
| **폰트** | 제목 18-24px/700, 본문 13-14px/500, 캡션 11-12px/400 |

## 6. 핵심 개선 우선순위

### TOP 5: 지금 당장 고쳐야 할 핵심

#### 1. 메뉴 통합: 공급사/셀러 DB 일원화 (긴급도: ★★★)
**현재:** 공급사 DB, 상품 DB, 상품 리스트 / 셀러 DB, 셀러 리스트
**개선:**
- `공급사 관리` 단일 메뉴 → 탭으로 "공급사 정보 | 상품 관리" 구분
- `셀러 관리` 단일 메뉴 → 탭으로 "셀러 정보 | 셀러 리스트" 구분
**효과:** 메뉴 복잡도 50% 감소, 클릭 동선 단축

#### 2. 공구-주문-정산 연결 강화 (긴급도: ★★★)
**현재:** 공구 화면과 주문 현황이 분리됨
**개선:**
- 공구 상세 화면에서 해당 공구의 주문 현황 바로 표시
- 정산 대기 공구를 공구 현황에서 바로 확인 가능
**효과:** 운영자가 공구별 상태를 한눈에 파악

#### 3. 버튼 스타일 통일 (긴급도: ★★☆)
**현재:** 화면마다 버튼 색상/스타일이 다름
**개선:**
```
기본 버튼: var(--gray-100) 배경
강조 버튼: var(--primary) 배경 (#fc9600)
위험 버튼: var(--danger) 배경 (#f04452)
성공 버튼: var(--success) 배경 (#10b981)
```
**효과:** 시각적 일관성으로 학습 비용 감소

#### 4. 글로벌 검색 기능 추가 (긴급도: ★★☆)
**현재:** 각 화면에서 개별 검색만 가능
**개선:**
- 상단 고정 검색창 추가
- 공급사/셀러/공구/상품 통합 검색
- 검색 결과에서 바로 해당 항목으로 이동
**효과:** 정보 접근 시간 80% 단축

#### 5. 정산 플로우 간소화 (긴급도: ★★☆)
**현재:** 엑셀 업로드 → 매핑 → 정산서 생성 (5단계)
**개선:**
- 공구 종료 시 자동으로 정산 대기 목록에 추가
- 원클릭 정산서 생성 버튼
- 정산 완료 체크만으로 완료 처리
**효과:** 정산 작업 시간 70% 단축

---

### 고도화하면 좋은 요소 (시간이 허락할 때)

#### 1. 통합 대시보드
- 운영센터 영업현황 + 전략센터 현황 대시보드 통합
- 한 화면에서 영업/전략 KPI 모두 확인

#### 2. 알림 시스템
- 정산 D+7 초과 시 자동 알림
- 체크리스트 미완료 항목 알림
- 신규 공구 제안 알림

#### 3. 모바일 최적화 강화
- 현재 FAB 메뉴 개선 (더 많은 퀵 액션)
- 테이블 → 카드 뷰 자동 전환
- 스와이프 제스처 지원

#### 4. 데이터 시각화 강화
- 공구 성공률 트렌드 차트
- 셀러별/공급사별 성과 비교
- 월별 매출 예측 그래프

#### 5. 역할별 대시보드
- 대표용: 전체 KPI 요약
- 담당자용: 내 담당 공구/파트너 중심
- 파트너용: 주문/정산 현황 중심

## 7. 메뉴 재구성 최종 제안

### 개선된 운영센터 메뉴 구조
```
📊 대시보드 (영업+전략 통합)

🛒 공구 관리
  ├─ 캘린더
  ├─ 공구 현황
  └─ 신규 제안

👥 파트너 관리
  ├─ 공급사 관리 (탭: 정보 | 상품)
  ├─ 셀러 관리 (탭: 정보 | 리스트)
  ├─ 계정 관리
  └─ 공지사항

💰 정산 관리
  ├─ 정산 대기
  ├─ 정산 완료
  └─ 정산서 관리

📋 업무 관리 (전략센터 통합)
  ├─ 전략기획
  ├─ KPI 관리
  ├─ 업무일지
  └─ 분기 리뷰

⚙️ 설정
  ├─ 팀원 관리
  ├─ 시스템 설정
  └─ 가이드
```

## 8. 결론

모여라딜 어드민 시스템은 공구 운영에 필요한 핵심 기능을 잘 갖추고 있으나, **메뉴 중복**, **화면 간 이동 복잡성**, **디자인 불일관성** 문제가 있습니다.

**최우선 개선:**
1. 공급사/셀러 메뉴 통합
2. 공구-주문-정산 연결 강화
3. 버튼/카드 스타일 통일

이 세 가지만 개선해도 운영자 경험이 크게 향상될 것입니다.
