# 모여라딜 어드민 시스템 - 종합 UX 리뷰 보고서

## 0. 시스템 개요 및 설계 원칙

### 0.1 현재 코드 규모
| 파일 | 코드 라인 | 비고 |
|------|----------|------|
| admin/index.html (운영센터) | **38,404줄** | 공구/파트너/정산 관리 |
| admin2/index.html (전략센터) | **37,405줄** | 전략/KPI/업무일지 관리 |

### 0.2 유지해야 할 현재 설계
| 설계 결정 | 유지 이유 |
|----------|----------|
| **운영센터/전략센터 분리** | 통합 시 7.5만줄 → 유지보수 불가, 현재 분리가 올바름 |
| **HTML 단일 파일 방식** | PWA 오프라인 지원, 배포 단순화 |
| **Firebase 직접 연동** | 백엔드 없이 빠른 개발/유지보수 가능 |

## 1. 전체 메뉴 구조 분석

### 1.1 현재 메뉴 구조

**운영센터 (admin/)**
```
📊 영업현황 ← 핵심 대시보드
공구센터
  ├─ 📅 캘린더
  ├─ ✅ 공구 현황/체크리스트
  └─ 💡 공구 제안 관리
파트너 관리
  ├─ 공급사 관리 (공급사 DB / 상품 DB / 상품 리스트)
  ├─ 셀러 관리 (셀러 DB / 셀러 리스트)
  ├─ 파트너 계정관리
  └─ 파트너 공지
🛒 카페24 주문 공유 [파트너용]
💰 카페24 정산 [정산서용]
  ├─ 1️⃣ 엑셀 업로드 & 매핑
  ├─ 2️⃣ 주문 관리
  └─ 3️⃣ 정산서 생성 & 관리
📚 업무 가이드
관리자 🔐
```

```
🎯 그로스보드
  ├─ 📖 사용 가이드
  ├─ ① 전략기획
  ├─ ② KPI 관리
  ├─ ③ 업무일지
  ├─ ④ 현황 대시보드
  └─ ⑤ 분기 리뷰
📚 레퍼런스
관리자 🔐
```

## 2. 메뉴 구조 문제점 및 개선안

### 2.1 개선 가능한 메뉴 중복

| 문제 | 현재 상태 | 개선안 | 유지보수 부담 |
|------|----------|--------|--------------|
| **공급사 DB vs 상품 DB vs 상품 리스트** | 3개 메뉴 분리 | 탭 UI로 통합 | 낮음 |
| **셀러 DB vs 셀러 리스트** | 2개 메뉴 분리 | 탭 UI로 통합 | 낮음 |

### 2.2 유지해야 할 분리 구조

| 현재 구조 | 유지 이유 |
|----------|----------|
| **운영센터/전략센터 분리** | 코드 복잡도 관리 (각 3.7만줄) |
| **영업현황 vs 현황 대시보드** | 센터별 역할이 다름, 데이터 연동으로 해결 |

### 2.3 빠진 기능 (운영 흐름 기준)

| 운영 단계 | 빠진 기능 | 필요 이유 | 구현 부담 |
|----------|----------|----------|----------|
| **공구 진행** | 공구별 주문 현황 요약 | 현재 카페24 주문과 분리됨 | 중간 |
| **정산** | 정산 예정 알림 | D+7 초과 시 놓칠 수 있음 | 중간 |
| **관리** | 센터 내 통합 검색 | 빠른 정보 접근 | 중간 |

### 2.4 메뉴 명칭 개선안

| 현재 명칭 | 문제점 | 개선안 | 변경 부담 |
|----------|--------|--------|----------|
| 공구 현황/체크리스트 | 이름이 길음 | `공구 관리` | 낮음 |
| 공구 제안 관리 | 주체 불명확 | `신규 공구 제안` | 낮음 |
| 카페24 주문 공유 | 내부 용어 노출 | `주문 현황` | 낮음 |

## 3. UX 관점 개선 사항 (유지보수 고려)

### 3.1 클릭 동선 개선

#### 문제 1: 운영센터 ↔ 전략센터 이동
**현재:** 사이드바 상단 링크 클릭 → 새 페이지 로딩
**개선안 (부담 낮음):**
```
센터 이동 링크에 context 파라미터 추가
../admin2/?from=settlement → 전략센터에서 관련 정보 하이라이트
```

#### 문제 2: 공구 생성 시 공급사/셀러 선택
**현재:** 공구 등록 → 공급사 드롭다운 → (없으면) 공급사 DB 이동
**개선안 (부담 중간):**
```
공구 등록 모달 내 "빠른 등록" 인라인 폼 추가
```

### 3.2 한 화면에서 함께 보여주면 좋은 정보

#### 제안 1: 공구 상세 화면에 요약 정보 추가 (부담 낮음)
현재 공구 모달에 주문/정산 요약만 추가:
```
┌─────────────────────────────────────────────────┐
│ 공구명: [ㅇㅇ 공구]                    D-Day: 3일차 │
├─────────────────────────────────────────────────┤
│ 📦 주문 요약: 152건 / 2,340,000원               │
│ 💰 정산 상태: 대기중 (D+5)                       │
│ ✅ 체크리스트: 3/5 완료                          │
└─────────────────────────────────────────────────┘
```
→ 기존 Firebase 데이터 활용, UI만 추가

#### 제안 2: 영업현황 카드 클릭 시 이동 (부담 낮음)
```
현재: 숫자만 표시
개선: onclick 이벤트로 해당 목록 화면 이동
```

### 3.3 자주 쓰는 기능 빠른 접근

#### 현재 모바일 FAB 활용 확장 (부담 낮음)
```javascript
// 현재 FAB 메뉴
공구 현황, 주문 관리, 정산, 영업 현황

// 데스크톱에도 상단 퀵 액션 추가 가능
[+ 새 공구] [💰 정산] [🔍 검색]
```

## 4. UI/디자인 불일관성 분석

### 4.1 컬러 시스템 현황

| 화면 | 현재 사용 컬러 | 브랜드 일관성 |
|------|--------------|--------------|
| 영업현황 | 파랑 그라디언트 (#1e40af → #3b82f6) | △ |
| 정산 관리 | 초록 그라디언트 (#059669 → #10b981) | △ |
| 주문 공유 | 파랑 그라디언트 (#2563eb → #3b82f6) | △ |
| **전략기획** | **주황 (#fc9600), 노랑 (#FEF3C7)** | **✓ 브랜드 일치** |
| 그로스보드 | 초록 (#10b981 → #059669) | △ 센터 구분용 |

### 4.2 버튼 스타일 불일관

**발견된 버튼 유형:**
1. `background: var(--gray-100)` - 회색 약한 버튼
2. `background: var(--gray-900)` - 검정 강한 버튼
3. `background: linear-gradient(...)` - 그라디언트 버튼 (색상 제각각)
4. `border: 2px solid ...` - 아웃라인 버튼

**개선 방향:** CSS 변수 활용으로 일괄 통일 (부담 낮음)

### 4.3 카드/섹션 스타일 불일관

| 스타일 | 사용 화면 |
|--------|----------|
| `border-radius: 16px` + `box-shadow` | 전략기획, 영업현황 |
| `border-radius: 12px` + `border: 1px solid` | 정산 관리 |
| `border-radius: 8px` | 일부 버튼, 입력 필드 |

## 5. 전략기획 메뉴 기준 디자인 가이드

### 5.1 레퍼런스 스타일 (전략기획 메뉴)

```css
/* 카드 기본 */
background: white;
border-radius: 16px;
padding: 24px;
box-shadow: 0 2px 8px rgba(0,0,0,0.06);

/* KPI 카드 */
background: linear-gradient(135deg, #fc9600, #e08600);
border-radius: 16px;
padding: 20px;
color: white;

/* 강조 섹션 (성공률 등) */
background: linear-gradient(135deg, #FEF3C7, #FDE68A);
border: 2px solid #F59E0B;
border-radius: 16px;

/* 프로그레스 바 */
background: var(--gray-200);
height: 5px;
border-radius: 3px;

/* 버튼 - 기본 */
background: var(--gray-100);
border: none;
padding: 10px 20px;
border-radius: 8px;
font-size: 13px;
font-weight: 600;

/* 버튼 - 강조 */
background: var(--primary);  /* #fc9600 */
color: white;
```

### 5.2 통일 적용 방법 (유지보수 친화적)

#### 방법 1: 스타일 상수 정의 (권장)
```javascript
// 각 render 함수 상단에 공통 스타일 상수
const STYLE = {
  card: 'background: white; border-radius: 16px; padding: 24px; box-shadow: 0 2px 8px rgba(0,0,0,0.06);',
  btnPrimary: 'background: var(--primary); color: white; border: none; padding: 10px 20px; border-radius: 8px; font-weight: 600;',
  btnSecondary: 'background: var(--gray-100); border: none; padding: 10px 20px; border-radius: 8px; font-weight: 600;'
};
```

#### 방법 2: CSS 변수 활용 강화
```css
/* 이미 정의된 변수들 */
--primary: #fc9600;
--gray-100 ~ --gray-900;
--success, --warning, --danger, --info;

/* 추가 정의 권장 */
--card-radius: 16px;
--btn-radius: 8px;
--card-shadow: 0 2px 8px rgba(0,0,0,0.06);
```

## 7. 현재 구조 유지하며 개선하는 방향

### 7.1 센터 분리 유지 + 연결 강화

```
[운영센터]                    [전략센터]
영업현황 ◄──── 데이터 공유 ────► 현황 대시보드
   │              (Firebase)           │
   │                                   │
공구 관리 ◄─── 퀵 링크 + 파라미터 ───► 전략기획
   │                                   │
정산 관리 ◄─── 정산 데이터 연동 ────► KPI 관리
```

### 7.2 코드 구조 개선 제안 (선택사항)

현재 단일 파일 구조를 유지하면서도:
```javascript

// ========== 공통 스타일 상수 ==========
const COMMON_STYLES = { ... };

// ========== 공통 유틸 함수 ==========

function formatDate(d) { ... }

// ========== 각 화면 렌더 함수 ==========
// (현재 구조 유지)
```

## 8. 결론

- **PWA 지원**: 오프라인/모바일 대응

2. 메뉴 명칭 직관화
3. 센터 간 퀵 링크 개선

### 개선 권장 (부담 중간)
4. 공급사/셀러 메뉴 탭 UI 통합
5. 공구 상세 화면 정보 보강
6. 센터 내 검색 기능

### 신중히 검토 (부담 높음)
7. 정산 플로우 재설계
8. 알림 시스템 도입
