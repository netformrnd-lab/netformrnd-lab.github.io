## 비즈니스 개요

### 플랫폼 구조

모여라딜은 **B2B2C 인플루언서 공구 플랫폼**입니다.

```
[공급사(브랜드)] → [모여라딜] → [셀러(인플루언서)] → [소비자]
       ↓                ↓                ↓              ↓
   제품 공급        공구 운영       콘텐츠 제작      공구 구매
```

### 수익 구조

| 주체 | 역할 | 수익 |
|------|------|------|
| **공급사** | 제품 제공, 재고 공급 | 공급가 × 판매 수량 |
| **모여라딜** | 공구 운영, 배송/CS | (판매가 - 공급가 - 셀러 수수료) × 판매 수량 |
| **셀러** | 콘텐츠 제작, SNS 판매 | 판매가 × 20~35% × 판매 수량 |
| **소비자** | 제품 구매 | 공구가 할인 + 레퍼럴 캐시백 |

### 주요 카테고리

- 뷰티/화장품
- 건강식품
- 식품/음료
- 생활용품
- 가전/디지털
- 패션/의류
- 육아용품
- 반려동물용품

## 공급사 관리 프로세스

### 1️⃣ 공급사 영입

**담당:** MD팀
**기간:** 1~2주
**목표:** 우수한 제품을 가진 브랜드/제조사 발굴 및 입점

#### 프로세스

```
[타겟 공급사 발굴] → [초기 컨택] → [통화/미팅] → [협상] → [입점신청] → [계약]
```

#### 세부 단계

**1단계: 타겟 공급사 발굴**
- **대상:** 연매출 1억 이상, 온라인 경험 있는 브랜드
- **우선순위:**
  - A등급: 플랫폼 타겟 정확히 맞는 제품, SNS 공구 경험 있는 브랜드
  - B등급: 제품력 좋으나 인지도 낮은 브랜드, 신제품 런칭 예정
  - C등급: 대형 브랜드 (경쟁 많음), 프리미엄 브랜드
- **조사 항목:**
  - 회사 규모 및 연혁
  - 주요 제품군 및 베스트셀러
  - 현재 유통 채널
  - 시장 평균 공급가

**2단계: 초기 컨택**
- **방법:** 이메일, 전화 콜드콜, SNS DM
- **도구:**
  - [공급사 제안 스크립트](../supplier/proposal-script.md)
  - [카테고리별 이메일 템플릿](../supplier/category/)
- **강조 사항:**
  - 초기비용 0원
  - 셀러 CS/배송 100% 대행
  - 신규 판로 개척
  - 안정적인 정산

**3단계: 통화 및 미팅**
- **통화 시간:** 5~10분
- **미팅 시간:** 30~40분
- **주요 설명:**
  - 모여라딜 플랫폼 소개
  - 협업 방식 (상품 등록 → 셀러 매칭 → 공구 진행 → 정산)
  - 수수료 구조 (공급가에 마진 포함, 별도 수수료 없음)
  - 성공 사례

**4단계: 협상**
- **핵심 협상 항목:**
  - **MOQ (최소주문수량)**
  - **공급가 (단가)**
  - **정산 주기**
  - **배송 방식** (위탁 입고 vs 직송)
- **협상 가이드:** [공급사 협상 가이드](../supplier/negotiation-guide.md)

**5단계: 입점신청**
- **URL:** [공급사 입점신청 폼](../supplier/apply/)
- **필수 정보:**
  - 회사명, 사업자등록번호
  - 담당자 이름, 연락처, 이메일
  - 제품명, 카테고리
  - 제품 이미지 URL
  - 공급가, 희망 판매가
  - MOQ

**6단계: 계약 체결**
- 공급 계약서 발송 및 서명
- 중요 조항: 공급가, 결제 조건, 정산 조건, 계약 기간

#### 후속 조치

- 통화/미팅 당일: 논의 내용 정리 이메일 발송
- 3일 후: 진행 의향 확인
- 계약 완료 후: 상품 등록 진행

#### 관련 문서

- [공급사 제안 스크립트](../supplier/proposal-script.md)
- [공급사 협상 가이드](../supplier/negotiation-guide.md)
- [카테고리별 제안 전략](../supplier/proposal-categories.md)

### 2️⃣ 공급사 상품 등록

**담당:** MD팀
**기간:** 1~3일
**시스템:** [공급사 포탈](../supplier/portal.html)

#### 프로세스

```
[상품 정보 수집] → [시스템 등록] → [이미지 준비] → [가격 책정] → [검수] → [등록 완료]
```

#### 세부 단계

**1단계: 상품 정보 수집**
- 제품명, 브랜드명
- 상세 설명 (제품 특징, 성분, 사용법)
- 제품 이미지 (대표 이미지, 상세 이미지)
- 카테고리 분류
- 공급가, 권장 소비자가

**2단계: 시스템 등록**
- 공급사 포탈에서 상품 등록
- 필수 필드 입력
- SKU 코드 생성

**3단계: 이미지 준비**
- 대표 이미지: 1장 (1:1 비율)
- 상세 이미지: 3~10장
- 셀러 공유용 이미지 제작 (스토리용, 피드용)

**4단계: 가격 책정**
- 공급가 확인
- 경쟁사 가격 조사
- 판매가 책정: 공급가 × 1.3 ~ 2.0
- 마진율 계산: (판매가 - 공급가 - 셀러 수수료) / 판매가

**5단계: 검수**
- MD팀 최종 확인
- 카테고리 적합성 검토
- 가격 경쟁력 확인

**6단계: 등록 완료**
- 공급사에 등록 완료 알림
- 샘플 요청 (셀러 매칭용)

### 3️⃣ 공급사 재고 관리

**담당:** 물류팀
**시스템:** [공급사 주문 현황](../supplier/orders.html)

#### 배송 방식

**방식 1: 위탁 입고**
- 공급사 → 모여라딜 창고 입고
- 모여라딜에서 주문 시 직접 출고
- 장점: 빠른 배송, 재고 통제 용이
- 단점: 창고 비용 발생

**방식 2: 직송**
- 공급사 → 소비자 직접 배송
- 모여라딜은 주문 데이터만 전달
- 장점: 창고 비용 없음
- 단점: 배송 시간 오래 걸림, 품질 통제 어려움

#### 재고 모니터링

- 공구 시작 전 재고 확보 확인
- 공구 진행 중 실시간 재고 업데이트
- 재고 부족 시 공급사에 긴급 연락
- 공구 종료 후 남은 재고 반품/보관 협의

---

### 4️⃣ 공급사 정산

**담당:** 재무팀
**기간:** 공구 종료 후 7일 이내
**시스템:** [카페24 정산 관리](../admin/)

#### 프로세스

```
[공구 종료] → [주문 집계] → [정산 금액 계산] → [세금계산서 요청] → [입금]
```

#### 세부 단계

**1단계: 공구 종료**
- 공구 기간 종료
- 반품/취소 처리 (7일 이내)

**2단계: 주문 집계**
- 카페24 주문 데이터 다운로드
- 취소/반품 제외한 확정 주문 수량 계산

**3단계: 정산 금액 계산**
- 정산 금액 = 공급가 × 판매 수량
- 배송비 정산 (별도 협의 시)

**4단계: 세금계산서 요청**
- 공급사에 세금계산서 발행 요청
- 세금계산서 확인

**5단계: 입금**
- 지정 계좌로 정산금 입금
- 입금 완료 알림

#### 정산 리포트 제공

- 판매 수량, 매출액
- 카테고리별 분석
- 베스트 셀러 분석
- 다음 공구 제안

## 셀러 관리 프로세스

### 1️⃣ 셀러 영입

**담당:** MD팀
**기간:** 3~7일
**목표:** 마이크로 인플루언서 발굴 및 입점

#### 프로세스

```
[타겟 셀러 발굴] → [SNS DM] → [통화] → [샘플 발송] → [테스트 공구] → [정식 등록]
```

#### 세부 단계

**1단계: 타겟 셀러 발굴**
- **이상적인 셀러 프로필:**
  - 팔로워: 1,000명 이상
  - 반응률: 게시물당 좋아요 50개 이상, 댓글 10개 이상
  - 콘텐츠: 제품 리뷰, 일상 공유, 정보성 콘텐츠
  - 활동빈도: 주 3회 이상 포스팅
  - 톤앤매너: 친근하고 신뢰감 있는 이미지
- **플랫폼:** 인스타그램, 블로그, 유튜브
- **카테고리:** 육아, 리빙, 식품, 뷰티

**2단계: 초기 컨택 (DM)**
- **방법:** 인스타그램 DM, 블로그 쪽지, 댓글
- **도구:** [셀러 제안 스크립트](../seller/proposal-script.md)
- **강조 사항:**
  - 샘플 무료 제공
  - 판매 수수료 업계 최고 수준 (20~35%)
  - 상세페이지/이미지 지원
  - CS 100% 대행

**3단계: 전화 통화**
- **통화 시간:** 5~10분
- **주요 설명:**
  - 모여라딜 플랫폼 소개
  - 수수료 구조
  - 셀러 역할 (콘텐츠 제작 + 링크 공유만)
  - 샘플 제공 및 테스트 공구 제안

**4단계: 입점신청**
- **URL:** [셀러 입점신청 폼](../seller/apply/)
- **필수 정보:**
  - 이름, 연락처, 이메일
  - SNS 계정 (인스타그램, 블로그 등)
  - 팔로워 수
  - 주요 카테고리
  - 배송지 주소 (샘플 수령용)

**5단계: 샘플 발송**
- 관심 카테고리 제품 샘플 발송
- 배송 추적 및 수령 확인

**6단계: 테스트 공구**
- 첫 공구는 소량으로 시작
- 콘텐츠 가이드 제공
- 판매 링크 제공
- 성과 모니터링

**7단계: 정식 등록**
- 테스트 공구 성공 시 정식 셀러 등록
- 전담 MD 배정
- 정기 공구 제안

#### 후속 조치

- 샘플 발송 2~3일 후: 수령 확인 및 사용 후기 요청
- 공구 진행 전: 공구 일정, 판매 링크, 공유용 이미지 제공
- 공구 종료 후: 판매 결과 공유, 정산 안내

#### 관련 문서

- [셀러 제안 스크립트](../seller/proposal-script.md)

### 2️⃣ 셀러 매칭

**담당:** MD팀
**기간:** 1~2일
**시스템:** [운영센터 - 파트너 관리](../admin/)

#### 프로세스

```
[신제품 입고] → [카테고리 분석] → [적합 셀러 선정] → [샘플 발송] → [공구 일정 조율]
```

#### 세부 단계

**1단계: 신제품 입고 알림**
- 공급사로부터 신제품 정보 수신
- 제품 특성, 타겟 고객층 분석

**2단계: 카테고리 분석**
- 제품 카테고리 확인 (예: 육아, 식품, 뷰티 등)
- 해당 카테고리 셀러 목록 확인

**3단계: 적합 셀러 선정**
- **선정 기준:**
  - 카테고리 일치
  - 팔로워 특성 (연령대, 관심사)
  - 최근 공구 성과
  - 콘텐츠 품질
- **우선순위:**
  - VIP 셀러 (월 5회 이상 또는 매출 상위)
  - 활성 셀러 (월 3회 이상)
  - 신규 셀러 (테스트 기회 제공)

**4단계: 샘플 발송**
- 선정된 셀러에게 샘플 발송
- 샘플 수량:
  - 신규 셀러: 1개
  - 활성 셀러: 2개
  - VIP 셀러: 무제한

**5단계: 공구 일정 조율**
- 셀러와 공구 시작일 협의
- 공구 기간: 보통 7일 (3~14일)
- 판매 목표 수량 설정

### 3️⃣ 셀러 지원

**담당:** MD팀, 디자인팀
**시스템:** [셀러 포탈](../seller/portal.html)

#### 제공 자료

**1. 제품 이미지**
- 대표 이미지 (정사각형, 인스타그램 피드용)
- 스토리용 이미지 (세로형, 9:16)
- 상세페이지 링크

**2. 제품 정보**
- 제품명, 브랜드명
- 주요 특징 (3~5줄)
- 가격 (정가, 공구가)
- 배송 정보
- 사용법, 주의사항

**3. 판매 링크**
- 공구 페이지 URL
- 레퍼럴 코드 (셀러별 고유)
- 짧은 URL (카카오톡 공유용)

**4. 콘텐츠 가이드**
- 권장 해시태그
- 주요 강조 포인트
- 금지 표현 (과장 광고, 의료 효능 등)

#### 셀러 교육

- 첫 공구 전: 공구 진행 방법 안내
- 주기적: 공구 성과 향상 팁 공유
- 정책 변경 시: 공지 및 안내

---

### 4️⃣ 셀러 정산

**담당:** 재무팀
**기간:** 공구 종료 후 7일 이내
**시스템:** [운영센터 - 정산 관리](../admin/)

#### 수수료 구조

| 등급 | 조건 | 기본 수수료 | 성과 보너스 |
|------|------|------------|------------|
| **신규 셀러** | 팔로워 1,000~5,000 | 20% | - |
| **일반 셀러** | 팔로워 5,000~10,000 또는 월 3회+ | 25% | +3% (월 10건 이상) |
| **우수 셀러** | 팔로워 10,000+ 또는 월 5회+ | 30% | +5% (월 20건 이상) |

#### 프로세스

```
[공구 종료] → [판매 집계] → [수수료 계산] → [통장 정보 확인] → [입금]
```

#### 세부 단계

**1단계: 공구 종료**
- 공구 기간 종료
- 반품/취소 처리 (7일 이내)

**2단계: 판매 집계**
- 셀러별 판매 수량 확인
- 레퍼럴 코드로 추적
- 취소/반품 제외

**3단계: 수수료 계산**
- 정산 금액 = (판매가 × 수수료율) × 판매 수량
- 예시:
  - 판매가 20,000원
  - 수수료 25%
  - 판매 수량 10개
  - 정산 금액 = 20,000 × 25% × 10 = 50,000원

**4단계: 통장 정보 확인**
- 셀러가 등록한 계좌 확인
- 예금주 일치 여부 확인

**5단계: 입금**
- 지정 계좌로 정산금 입금
- 입금 완료 알림

#### 정산 리포트 제공

- 판매 수량, 매출액
- 수수료 금액
- 성과 분석 (이전 공구 대비)
- 다음 공구 추천

## 공구 진행 프로세스

### 전체 프로세스

```
[공구 기획] → [상품 등록] → [셀러 매칭] → [샘플 검수] → [공구 시작] → [판매] → [종료] → [정산]
```

### 1️⃣ 공구 기획

**담당:** MD팀
**기간:** 3~7일
**시스템:** [운영센터 - 공구 관리](../admin/)

#### 기획 항목

**1. 공구 목표 설정**
- 판매 목표 수량
- 예상 매출
- 타겟 고객층

**2. 제품 선정**
- 신제품 런칭
- 시즌 상품 (예: 여름 시즌 선크림)
- 재고 소진 필요 상품
- 고객 요청 많은 상품

**3. 가격 책정**
- 공급가 확인
- 경쟁사 가격 조사
- 공구가 결정 (일반가 대비 20~50% 할인)
- 마진율 계산

**4. 공구 기간 결정**
- 단기 공구: 3~5일 (한정 수량, 긴박감 조성)
- 일반 공구: 7일 (가장 일반적)
- 장기 공구: 10~14일 (고가 제품, 신중한 구매)

**5. 프로모션 기획**
- 첫 구매 할인
- 묶음 할인 (2+1, 3+1 등)
- 레퍼럴 캐시백
- 체험단 모집

```
[주문 접수] → [주문 확인] → [배송 준비] → [포장] → [배송] → [배송 완료] → [구매 확정]
```

## CS 처리 프로세스

### 전체 프로세스

```
[문의 접수] → [분류] → [1차 응답] → [처리] → [완료]
```

### 1️⃣ 문의 접수

**채널:**
- 카카오톡 채널
- 이메일
- 전화
- 공구 페이지 1:1 문의
- SNS DM (셀러가 모여라딜로 전달)

**시스템:** [운영센터 - CS 관리](../admin/)

---

### 2️⃣ 문의 분류

**담당:** CS팀

#### 문의 유형

| 유형 | 내용 | 우선순위 | 목표 응답 시간 |
|------|------|---------|--------------|
| **주문 관련** | 주문 변경, 취소, 추가 | 높음 | 1시간 이내 |
| **배송 관련** | 배송 조회, 배송지 변경 | 높음 | 1시간 이내 |
| **제품 문의** | 성분, 사용법, 효과 | 보통 | 3시간 이내 |
| **교환/반품** | 교환, 반품, 환불 | 높음 | 1시간 이내 |
| **불량/하자** | 제품 불량, 파손 | 긴급 | 즉시 |
| **기타** | 일반 문의 | 낮음 | 24시간 이내 |

---

### 3️⃣ 문의 처리

#### 주문 관련 CS

**주문 변경**
- 배송 준비 전: 변경 가능
- 배송 준비 중/배송 시작 후: 변경 불가
- 처리: 주문 시스템에서 직접 수정

**주문 취소**
- 배송 준비 전: 즉시 취소 가능
- 배송 시작 후: 취소 불가 (반품 안내)
- 처리: 주문 시스템에서 취소 처리 → 환불

---

#### 배송 관련 CS

**배송 조회**
- 배송 번호 안내
- 택배사 추적 링크 제공

**배송지 변경**
- 배송 시작 전: 변경 가능
- 배송 시작 후: 택배사 고객센터로 직접 연락 안내

**배송 지연**
- 택배사 확인
- 지연 사유 안내
- 예상 도착일 안내

**배송 사고** (분실, 파손)
- 택배사에 사고 접수
- 재발송 또는 환불 처리

---

#### 제품 문의 CS

**성분/사용법 문의**
- 제품 상세페이지 정보 안내
- 공급사 제공 자료 전달
- 복잡한 문의는 공급사에 확인 후 응답

**효과/효능 문의**
- 객관적 정보 제공
- 과장 광고 금지
- 의료 효능 문의는 전문가 상담 권장

---

#### 교환/반품 CS

**교환/반품 가능 조건**
- 배송 완료 후 7일 이내
- 제품 미개봉, 상품 태그 훼손 없음
- 단순 변심: 왕복 배송비 고객 부담
- 제품 불량/오배송: 왕복 배송비 판매자 부담

**교환/반품 프로세스**
```
[교환/반품 신청] → [사유 확인] → [반품 주소 안내] → [상품 회수] → [검수] → [교환/환불 처리]
```

**교환/반품 처리**
1. 교환/반품 신청서 작성
2. 반품 주소 안내
3. 고객이 상품 반송
4. 상품 수령 및 검수
5. 교환: 새 상품 재발송
6. 반품: 환불 처리 (카드 취소 또는 계좌 입금)

---

#### 불량/하자 CS (긴급)

**처리 프로세스**
```
[불량 접수] → [사진 요청] → [공급사 확인] → [즉시 재발송 또는 환불]
```

**세부 단계**
1. 불량 사진/동영상 요청
2. 공급사에 불량 사진 전달 및 확인
3. 공급사 귀책: 즉시 재발송 (배송비 공급사 부담)
4. 공급사가 재발송 불가 시: 전액 환불
5. 불량 상품 회수 (필요 시)

---

### 4️⃣ CS 완료 및 피드백

**완료 기준**
- 고객 문제 해결 완료
- 고객 만족도 확인
- 추가 문의 없음

**피드백 수집**
- CS 만족도 조사 (선택)
- 불만 사항 수집
- 개선 사항 도출

**통계 관리**
- 일일 CS 건수
- 유형별 분류
- 평균 응답 시간
- 해결율
- 개선 방안 도출

### 2️⃣ 정산 금액 계산

#### 공급사 정산

**정산 금액 = 공급가 × 최종 판매 수량**

**예시:**
- 공급가: 10,000원
- 최종 판매 수량: 100개
- 정산 금액: 10,000 × 100 = 1,000,000원

#### 셀러 정산

**정산 금액 = (판매가 × 수수료율) × 최종 판매 수량**

**예시:**
- 판매가: 15,000원
- 수수료율: 25%
- 셀러별 판매 수량: 30개
- 정산 금액: (15,000 × 25%) × 30 = 112,500원

## 레퍼럴 캐시백 프로세스

> **참고:** 레퍼럴 캐시백 시스템은 현재 개발 계획 단계입니다.
> 자세한 내용은 [레퍼럴 캐시백 설계서](../REFERRAL_CASHBACK_DESIGN.md)를 참고하세요.

### 전체 프로세스

```
[레퍼럴 링크 생성] → [친구 공유] → [친구 구매] → [캐시백 적립] → [캐시백 사용]
```

---

### 1️⃣ 레퍼럴 링크 생성

**담당:** 고객 (자동화)
**시스템:** 공구 페이지 - 레퍼럴 시스템

#### 프로세스

1. 고객이 공구 페이지 접속
2. "친구 초대" 버튼 클릭
3. 고유 레퍼럴 코드 생성 (예: ABC123XY)
4. 레퍼럴 링크 생성: `https://moyeoradeal.com/deals/view?id=deal456&ref=ABC123XY`
5. 공유 옵션 제공 (카카오톡, SNS, 링크 복사)

---

### 2️⃣ 친구 공유 및 클릭 추적

**시스템:** 레퍼럴 추적 시스템

#### 프로세스

1. 추천인이 레퍼럴 링크 공유 (카카오톡, SNS, 문자)
2. 피추천인이 레퍼럴 링크 클릭
3. 쿠키에 레퍼럴 코드 저장 (유효기간: 30일)
4. 클릭 수 증가 (Firestore: referrals 컬렉션)

---

### 3️⃣ 친구 구매 및 레퍼럴 추적

**시스템:** 카페24 주문 연동

#### 프로세스

1. 피추천인이 공구 페이지에서 "구매하기" 버튼 클릭
2. 카페24 쇼핑몰로 리다이렉트
3. 주문서에 레퍼럴 코드 자동 입력 (주문 메모 또는 사용자 정의 필드)
4. 피추천인이 결제 완료
5. 주문 완료 후 카페24 → 모여라딜 주문 동기화

---

### 4️⃣ 캐시백 적립

**담당:** 자동화 (Cloud Functions)
**기간:** 주문 동기화 후 즉시

#### 적립 금액

- **추천인 캐시백:** 5,000원
- **피추천인 캐시백:** 3,000원

#### 프로세스

1. 주문 동기화 시 레퍼럴 코드 인식
2. 레퍼럴 코드 → 추천인 식별
3. 캐시백 적립:
   - 추천인 캐시백: 5,000원 적립
   - 피추천인 캐시백: 3,000원 적립
4. Firestore `cashbacks` 컬렉션에 기록
5. 적립 알림 발송 (카카오톡, 이메일)

#### 캐시백 데이터

- userId: 사용자 ID
- amount: 캐시백 금액
- earnedAt: 적립 일시
- expiresAt: 만료 일시 (적립일 + 1년)
- status: 'active' (사용 가능)
- usedAt: null

---

### 5️⃣ 캐시백 조회

**시스템:** 사용자 대시보드

#### 조회 정보

- 총 캐시백 잔액
- 사용 가능 캐시백
- 사용 내역
- 만료 예정 캐시백

---

### 6️⃣ 캐시백 사용

**시스템:** 카페24 체크아웃, 모여라딜 공구 페이지

#### 사용 프로세스

1. 고객이 다음 공구 제품 구매
2. 결제 시 "캐시백 사용" 선택
3. 사용할 캐시백 금액 입력 (최대 잔액까지)
4. 결제 금액 차감: 상품가 - 캐시백
5. 사용된 캐시백 처리:
   - status: 'active' → 'used'
   - usedAt: 사용 일시
   - orderId: 사용된 주문번호
6. 결제 완료

#### 사용 규칙

- **FIFO 방식:** 먼저 적립된 캐시백부터 사용
- **최소 주문 금액:** 10,000원 이상 주문 시 사용 가능
- **최대 사용 금액:** 주문 금액의 50%까지
- **유효기간:** 적립일로부터 1년

## 업무 개선 제안

### 자동화 필요 영역

1. **셀러 매칭 자동화**
   - 현재: MD가 수동으로 셀러 선정
   - 개선: 카테고리, 팔로워 특성, 과거 성과 기반 자동 추천

2. **샘플 관리 자동화**
   - 현재: 수동으로 샘플 발송 관리
   - 개선: 샘플 신청 → 승인 → 발송 추적 자동화

3. **정산 자동화**
   - 현재: 엑셀 다운로드 → 수동 계산 → 수동 입금
   - 개선: 주문 동기화 → 자동 계산 → API 입금

4. **CS 자동 응답**
   - 현재: 수동 응답
   - 개선: FAQ 기반 챗봇 자동 응답

### 분석 고도화

1. **카테고리별 성과 분석**
   - 카테고리별 판매량, 매출, 마진율
   - 시즌별 트렌드 분석

2. **셀러별 성과 분석**
   - 셀러별 판매 성과, ROI
   - 우수 셀러 발굴 및 인센티브

3. **고객 분석**
   - 재구매율, LTV (Life Time Value)
   - 레퍼럴 전환율
   - 고객 세그먼트 분석
