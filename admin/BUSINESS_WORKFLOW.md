## 시스템 개요

### 메뉴 구조

```
📁 업무관리
├─ 📅 달력 - 공구 캘린더 보기
├─ 📋 딜관리 - 공구 목록 및 관리
├─ 🏢 딜센터 - 공구 제안 관리
├─ ❓ 공급사 문의 - 공급사로부터 받은 문의
└─ 📦 주문 상태 - 주문 현황 확인

📁 파트너관리
├─ 🏭 공급사 - 공급사 목록 및 등록
├─ 👤 셀러 - 셀러 목록 및 등록
├─ 🔗 공유 링크 - 파트너 공유 링크 생성
└─ 📢 파트너 공지 - 공급사/셀러에게 공지

📁 정산관리
├─ 📊 카페24 매핑 - 카페24 주문 매핑 (가장 중요!)
├─ 📥 카페24 주문 - 카페24 주문 내역 조회
└─ 💰 정산 - 셀러/공급사 정산 관리
```

### 주요 데이터베이스 구조

| 컬렉션명 | 용도 |
|---------|------|
| `deals` | 공구/이벤트 정보 |
| `suppliers` | 공급사 정보 |
| `sellers` | 셀러 정보 |
| `supplierProducts` | 공급사 상품 DB |
| `cafe24_orders` | 카페24 주문 데이터 |
| `sellerSettlements` | 셀러 정산 관리 |
| `supplierSettlements` | 공급사 정산 관리 |

## 공구 진행 워크플로우

### 전체 프로세스

```
[공구 등록] → [여정 관리] → [카페24 매핑] → [정산 처리] → [완료]
```

---

### 1단계: 공구 등록하기

#### 화면 경로
**업무관리 > 📋 딜관리**

#### 등록 절차

**1) "공구 등록" 버튼 클릭**
- 우측 상단의 `+ 공구 등록` 버튼 클릭
- 공구 등록 모달 팝업

**2) 공급사 선택**
- `공급사` 드롭다운 클릭
- 공급사 목록에서 선택
- 📌 공급사가 없으면 먼저 **파트너관리 > 🏭 공급사**에서 등록 필요

**3) 셀러 선택**
- `셀러` 드롭다운 클릭
- 셀러 목록에서 선택
- 📌 셀러가 없으면 먼저 **파트너관리 > 👤 셀러**에서 등록 필요

**4) 공구명 자동 생성 확인**
- 공급사와 셀러를 선택하면 자동으로 `공구명` 필드에 생성됨
- 예: `브랜드A x 셀러B 공구`
- 필요 시 수동으로 수정 가능

**5) 카페24 상품번호 입력**
- 카페24 쇼핑몰에서 등록한 상품번호 입력
- 예: `12345`
- 📌 나중에 카페24 매핑 시 이 번호로 자동 매칭됨

**6) 공구 기간 설정**
- `시작일` 캘린더에서 선택
- `종료일` 캘린더에서 선택
- 일반적으로 7일 권장 (3~14일)

**7) 상품 선택**
- 공급사 선택 후 해당 공급사의 상품 목록이 표시됨
- 체크박스로 상품 선택 (다중 선택 가능)
- 각 상품별 **마진율(%)** 입력
  - 예: `30%` → 공급가가 10,000원이면 판매가는 13,000원
- 📌 상품이 없으면 **파트너관리 > 🏭 공급사**에서 상품 추가 필요

**8) 상태 선택**
- `협의중` (기본값) - 공급사/셀러와 협의 중
- `협의완료` - 공구 확정됨

**9) 저장**
- `저장` 버튼 클릭
- 공구가 `deals` 컬렉션에 저장됨
- 공구 목록에 표시됨

### 2단계: 여정 체크리스트 관리

#### 화면 경로
**업무관리 > 📋 딜관리 > 공구 클릭**

#### 여정 단계

공구 수정 모달에서 **여정 체크리스트** 섹션을 확인할 수 있습니다.

**준비 단계**
- ☐ **샘플 전달** - 셀러에게 제품 샘플 발송 완료
- ☐ **결제링크 전달** - 카페24 결제 링크 셀러에게 공유
- ☐ **콘텐츠 전달** - 제품 이미지, 상세페이지 전달
- ☐ **광고 세팅** - SNS 광고 설정 (필요 시)
- ☐ **셀러 가이드** - 공구 진행 가이드 전달

**진행 단계**
- ☐ **주문 모니터링** - 실시간 주문 현황 확인
- ☐ **고객 문의 대응** - CS 처리

**종료 단계**
- ☐ **발주서 전달** - 공급사에 발주서 전달
- ☐ **셀러 정산** - 셀러 정산 완료
- ☐ **공급사 정산** - 공급사 정산 완료
- ☐ **완료** - 공구 종료

#### 체크 방법
1. 공구 목록에서 공구 클릭
2. 공구 수정 모달에서 해당 항목 체크
3. `저장` 버튼 클릭

### 3단계: 카페24 주문 매핑 (중요!)

#### 화면 경로
**정산관리 > 📊 카페24 매핑**

#### 카페24 매핑이란?
카페24 쇼핑몰에서 발생한 주문을 모여라딜의 공구/셀러/공급사와 자동으로 연결하는 프로세스입니다.

#### 매핑 절차

**STEP 1: 카페24에서 주문 엑셀 다운로드**

1. 카페24 관리자 접속
2. **주문관리 > 전체주문조회**
3. 기간 설정 (예: 2026-01-01 ~ 2026-01-31)
4. **엑셀 다운로드** 버튼 클릭
5. 엑셀 파일 다운로드 완료

**STEP 2: 운영센터에서 엑셀 업로드**

1. 운영센터에서 **정산관리 > 📊 카페24 매핑** 클릭
2. **카페24 매핑 모달** 팝업
3. **STEP 1: 엑셀 업로드** 섹션에서 파일 선택
4. 다운로드한 엑셀 파일 선택
5. `업로드` 버튼 클릭
6. 파싱 완료 메시지 확인

**STEP 2에서 자동으로 처리되는 것들:**
- 주문번호, 상품번호, 주문날짜 파싱
- 공급사명 자동 추출
- 셀러명 자동 추출 (상품명에서 추출)
- 상품DB에서 공급단가/마진율 자동 매칭
- PG수수료 자동 계산 (신용카드 4%)

**STEP 3: 자동 분류 & 공구 매핑**

업로드가 완료되면 자동으로 다음 단계로 넘어갑니다:

- 주문 데이터가 **상품코드별**로 그룹화됨
- 각 그룹에 대해:
  - 셀러명으로 완료된 공구 필터링
  - 상품명 유사도 검사로 공구 자동 매칭
  - 매칭 실패 시 최근 종료된 공구 자동 선택

**자동 매칭 결과 확인:**

| 상품코드 | 상품명 | 셀러 | 수량 | 매출 | 매칭된 공구 | 확정 |
|---------|-------|------|------|------|----------|------|
| P001 | 샴푸 x 유미니89 | 유미니89 | 50개 | 750,000원 | ✅ 샴푸 공구 (자동) | ☐ |
| P002 | 로션 x 뷰티맘 | 뷰티맘 | 30개 | 450,000원 | ⚠️ 수동확인 필요 | ☐ |

**확정 처리:**

1. 자동 매칭된 항목 확인
2. 수동확인 필요한 항목은 드롭다운에서 공구 선택
3. 각 항목의 `확정` 체크박스 체크
4. 또는 `전체 자동확정` 버튼 클릭 (수동확인 제외 전체 확정)

**STEP 4: 일괄 적용**

1. 확정된 항목들을 확인
2. `일괄 적용` 버튼 클릭
3. 모든 주문 데이터가 `cafe24_orders` 컬렉션에 저장됨
4. 완료 메시지 확인

**저장되는 데이터:**
- 주문번호, 상품번호, 주문날짜
- 공급사명, 셀러명
- 수량, 단가, 총액
- 공급단가, 마진율
- 매칭된 공구ID
- PG수수료

### 4단계: 정산 처리

#### 화면 경로
**정산관리 > 💰 정산**

#### 정산 종류

**1) 셀러 정산**

화면: **셀러 정산** 탭

**자동 정산 (카페24 연동)**
- 카페24 매핑이 완료되면 자동으로 정산 데이터 생성됨
- 셀러별, 월별로 집계됨
- `완료` 버튼 클릭하면 정산 완료 처리

**수동 정산 (필요 시)**
1. `+ 새 정산 생성` 버튼 클릭
2. 정산 정보 입력:
   - 셀러명
   - 공급사명
   - 정산기간 (YYYY-MM)
   - 상품유형 (자사/타사)
   - 판매건수
   - 총 판매액
   - 총 마진금
3. `저장` 버튼 클릭

**정산 완료 처리**
1. 정산 목록에서 정산 항목 클릭
2. `완료` 버튼 클릭
3. 정산완료일이 자동으로 기록됨
4. PDF 다운로드 가능

**2) 공급사 정산**

화면: **공급사 정산** 탭

**자동 정산 (카페24 연동)**
- 카페24 매핑이 완료되면 자동으로 정산 데이터 생성됨
- 공급사별, 월별로 집계됨

**수동 정산 (필요 시)**
1. `+ 새 정산 생성` 버튼 클릭
2. 정산 정보 입력:
   - 공급사명
   - 정산기간
   - 총액
   - 주문건수
3. `저장` 버튼 클릭

**정산 완료 처리**
1. 정산 목록에서 정산 항목 클릭
2. `완료` 버튼 클릭
3. 정산완료일이 자동으로 기록됨

## 공급사 온보딩 워크플로우

### 전체 프로세스

```
[웹 입점신청] → [신청서 검토] → [운영센터 등록] → [서류 수집] → [상품 등록] → [계약 완료]
```

---

### 0단계: 웹 입점신청 (공급사가 직접 신청)

#### 화면 경로
**https://moyeoradeal.com/supplier/apply/**

#### 입점신청 폼 구조

**레이아웃**
- 왼쪽: 검은색 배경 + 슬라이더 (자동 애니메이션 25초 루프)
- 오른쪽: 흰색 입력 폼 영역 (최대 480px, 스크롤 가능)

#### 입력 필드

**1) 기본 정보 섹션**

| 필드명 | 타입 | 필수 | 설명 |
|-------|------|------|------|
| 기업명 | text | ✓ | 회사명 입력 |
| 자사몰 or 홈페이지 | text | ✗ | URL 입력 (예: https://...) |
| 담당자 이름 / 직급 | text | ✓ | 예: 홍길동 / 대리 |
| 연락처 | tel | ✓ | 010-0000-0000 (자동 포맷팅) |

**2) 파일 첨부 섹션**

| 필드명 | 타입 | 필수 | 제약사항 |
|-------|------|------|---------|
| 회사소개서 / 제안서 첨부 | file | ✗ | PDF, DOC, DOCX, PPT, PPTX, HWP, ZIP / 10MB 이하 |

**파일 업로드 프로세스:**
1. `파일 선택` 버튼 클릭
2. 파일 선택 (10MB 이하)
3. 자동 Firebase Storage 업로드 시작
4. 진행률 표시 (프로그레스 바 + %)
5. 업로드 완료 시 "업로드 완료!" 메시지

**3) 개인정보 동의**

- 개인정보처리방침 동의 (필수 체크)
- [약관확인] 링크로 내용 확인 가능

#### 제출 및 저장

**제출 버튼:** `신청하기`

**저장 위치:**
- Firebase Firestore: `supplierApplications` 컬렉션
- Google Sheets: 레거시 호환용

**저장 데이터:**
```javascript
{
  companyName: "기업명",
  website: "자사몰/홈페이지",
  managerName: "담당자 이름/직급",
  phone: "010-0000-0000",
  fileName: "업로드 파일명",
  fileUrl: "Firebase Storage URL",
  status: "pending",
  createdAt: "타임스탬프",
  source: "web_application_v3"
}
```

**제출 완료:**
- "✅ 입점신청이 완료되었습니다!" 모달 표시
- "영업일 기준 1~2일 내에 담당자가 연락드리겠습니다" 안내
- [확인] 버튼 클릭 시 /supplier/로 리다이렉트

### 1단계: 운영센터에서 공급사 등록

#### 화면 경로
**운영센터 > 파트너관리 > 🏭 공급사**

#### 등록 절차

**1) "공급사 등록" 버튼 클릭**
- 우측 상단의 `+ 공급사 등록` 버튼 클릭

**2) 기본정보 입력**

- **공급사명** (필수)
  - 예: `브랜드A 주식회사`
- **유형** 선택
  - `자사` - 모여라딜이 직접 운영
  - `타사` - 외부 브랜드
- **기업규모**
  - 예: `중소기업`, `중견기업`, `대기업`
- **홈페이지**
  - 예: `https://brand-a.com`

**3) 담당자 정보 입력**

- **담당자명**: 공급사 담당자 이름
- **직급**: 예: `과장`, `팀장`
- **연락처**: 예: `010-1234-5678`
- **이메일**: 예: `contact@brand-a.com`

**4) 상품 문의 방안**
- 공급사에 상품을 문의하는 방법 기록
- 예: `카카오톡 채널`, `이메일`, `담당자 직통`

**5) 계약 상태**
- `계약 대기중` (기본값)
- `계약 완료`

**6) 통장 정보**
- 정산을 위한 계좌번호 입력

**상품홈페이지 링크**
- 필드명: `productHomepageLink`
- 용도: 특정 상품 페이지 URL
- 예: `https://brand-a.com/products/shampoo`
- 입력 위치: 상품 수정 모달 > "상품홈페이지" 입력란
- 표시: 이모지 🔗 링크 (클릭 시 새탭 열기)

**링크 입력 방법:**
1. 상품 리스트에서 상품 클릭
2. 상품 수정 모달 팝업
3. "홈페이지" 필드에 URL 입력 (예: https://example.com)
4. "상품홈페이지" 필드에 URL 입력 (예: https://example.com/product)
5. `수정하기` 버튼 클릭

**자동 URL 처리:**
- http:// 또는 https://로 시작하지 않으면 자동으로 https:// 추가
- 예: `example.com` 입력 → `https://example.com`으로 저장

**사용 방법:**

#### 다운로드 절차

1. 카페24 주문조회 화면에서
2. **엑셀 다운로드** 버튼 클릭
3. 파일 다운로드 완료
4. 파일명 예: `order_202601.xlsx`

#### 필수 컬럼 확인

다운로드한 엑셀에 다음 컬럼이 있는지 확인:
- 주문번호
- 상품번호
- 자체상품코드
- 상품명(관리용)
- 상품명(기본)
- 상품명(옵션포함)
- 상품구매금액(KRW)
- 총주문금액(KRW)
- 결제상태 (F=미결제, T=결제완료)
- 주문일

### 3단계: 운영센터에서 매핑

#### 화면 경로
**정산관리 > 📊 카페24 매핑**

#### 매핑 절차

**STEP 1: 엑셀 업로드**

1. `파일 선택` 버튼 클릭
2. 다운로드한 엑셀 파일 선택
3. `업로드` 버튼 클릭
4. "파싱 완료: X건" 메시지 확인

**업로드 시 자동 처리:**
- 주문 데이터 파싱
- 셀러명 추출 (상품명에서)
- 공급사명 추출 (자체상품코드에서)
- 상품DB 매칭 (공급단가, 마진율)
- PG수수료 계산

**STEP 2: 자동 분류 결과 확인**

업로드가 완료되면 자동으로 **STEP 2**로 이동합니다.

**화면 구성:**

| 상품코드 | 상품명 | 공급사 | 셀러 | 수량 | 판매액 | 공급단가 | 마진율 | 매칭 공구 | 확정 |
|---------|-------|-------|------|------|-------|---------|-------|---------|------|
| P001 | 샴푸 | 브랜드A | 유미니89 | 50 | 750,000 | 10,000 | 30% | ✅ 샴푸 공구 | ☐ |
| P002 | 로션 | 브랜드B | 뷰티맘 | 30 | 450,000 | 8,000 | 25% | ⚠️ 수동 | ☐ |

**자동 매칭 로직:**
1. 셀러명으로 완료된 공구 필터링
2. 상품명 유사도 검사
3. 매칭 성공: ✅ 공구명 표시
4. 매칭 실패: ⚠️ 수동확인 필요 (드롭다운에서 선택)

**수동 확인이 필요한 경우:**
- 셀러가 여러 공구를 진행한 경우
- 상품명이 달라서 자동 매칭 실패

**수동 매칭 방법:**
1. ⚠️ 표시된 항목의 드롭다운 클릭
2. 공구 목록에서 올바른 공구 선택
3. `확정` 체크박스 체크

**전체 자동확정:**
- `전체 자동확정` 버튼 클릭
- 수동확인 필요한 항목 제외하고 전체 확정

**STEP 3: 정산 미리보기**

확정된 항목들의 정산 금액이 미리 계산됩니다.

**셀러별 정산 미리보기:**

| 셀러명 | 공급사 | 판매건수 | 총 판매액 | 총 마진금 | 정산 예정액 |
|-------|-------|---------|----------|----------|----------|
| 유미니89 | 브랜드A | 50 | 750,000 | 225,000 | 217,575 |
| 뷰티맘 | 브랜드B | 30 | 450,000 | 112,500 | 108,900 |

**정산 예정액 계산:**
```
총 마진금 - 셀러 원천징수(3.3%)
= 225,000 - (225,000 × 3.3%)
= 217,575원
```

**공급사별 정산 미리보기:**

| 공급사 | 주문건수 | 총 공급액 | 정산 예정액 |
|-------|---------|----------|----------|
| 브랜드A | 50 | 500,000 | 500,000 |
| 브랜드B | 30 | 240,000 | 240,000 |

**STEP 4: 일괄 적용**

1. 정산 미리보기 확인
2. `일괄 적용` 버튼 클릭
3. 다음이 자동으로 처리됨:
   - `cafe24_orders` 컬렉션에 주문 데이터 저장
   - `sellerSettlements` 컬렉션에 셀러 정산 생성
   - `supplierSettlements` 컬렉션에 공급사 정산 생성
   - `cafe24_settlements` 컬렉션에 정산 기록 저장
4. "매핑 완료" 메시지 확인

### 4단계: 정산 완료 처리

#### 화면 경로
**정산관리 > 💰 정산**

#### 셀러 정산 완료

1. **셀러 정산** 탭 선택
2. 정산 목록에서 정산 항목 확인
3. `완료` 버튼 클릭
4. 정산완료일 자동 기록
5. 상태가 `완료`로 변경됨
6. 필요 시 `PDF 다운로드` 버튼으로 정산서 다운로드

#### 공급사 정산 완료

1. **공급사 정산** 탭 선택
2. 정산 목록에서 정산 항목 확인
3. `완료` 버튼 클릭
4. 정산완료일 자동 기록
5. 상태가 `완료`로 변경됨

## 자주 하는 질문 (FAQ)

### Q1: 공구 등록 시 공급사/셀러가 없어요
**A:** 먼저 **파트너관리 > 🏭 공급사** 또는 **👤 셀러**에서 등록하세요.

### Q2: 카페24 매핑에서 공구가 자동 매칭이 안 돼요
**A:** 드롭다운에서 수동으로 공구를 선택하고 `확정` 체크하세요.

### Q3: 정산 금액이 안 맞아요
**A:** 상품DB에 공급단가와 마진율이 정확히 입력되었는지 확인하세요.

### Q4: 엑셀 업로드가 안 돼요
**A:** 카페24에서 다운로드한 원본 엑셀 파일인지 확인하세요. 편집하지 마세요.

### Q5: 공구 삭제는 어떻게 하나요?
**A:** 공구 수정 모달 하단의 `삭제` 버튼을 클릭하세요. (관리자 권한 필요)
